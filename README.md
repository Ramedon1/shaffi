# shaffi — eBPF Traffic Shaper

Шейпер трафика на базе eBPF + EDT (Earliest Departure Time) для Linux-серверов.

Работает напрямую в ядре, ограничивает скорость per-IP без коллизий, поддерживает IPv4/IPv6 и до 32 независимых правил на разные порты.

**Требования:** Linux 5.4+, `clang`, `bpftool`, `libbpf-dev`

---

## Установка

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/your-org/shaffi/main/install.sh)
```

Скрипт скачает все файлы в `/opt/shaffi` и создаст системную команду `shaffi`.

После установки:

```bash
shaffi
```

Откроется TUI-меню. Далее всё через интерфейс: выбор интерфейса, режима, портов и скоростей.

---

## Как работает

Шейпер прикрепляется к сетевому интерфейсу через `tc clsact` и перехватывает каждый пакет на уровне ядра.

- **Download** — управляется через EDT: каждому IP выставляется время отправки пакета, из-за чего `fq` qdisc естественно выравнивает скорость. Без сброса пакетов, без jitter.
- **Upload** — управляется через Token Bucket с hard drop: лишние пакеты сбрасываются, TCP сам снижает скорость отправителя.

Данные о каждом IP хранятся в BPF hash map (`user_state_map`). Конфигурация правил — в `config_map` и `port_rule_map`.

---

## Режимы

**Статический** — жёсткий лимит на каждый IP в любой момент времени.

**Динамический (Burst + Штраф)** — пользователь качает на полной скорости до исчерпания квоты за окно N секунд, после чего скорость снижается до штрафной на заданное время. Обычный пользователь (соцсети, VoIP) квоту не превышает и штрафа не получает.

**Shared Pool** — единый канал на всех пользователей порта. Если трубка 100 МБ/с занята одним — он качает 100, если десятью — по 10 каждый.

---

## Мультиправила

До 32 независимых правил одновременно, каждое на своих портах:

```
Правило #0 | Порт 443  | Статика   | DL/UL 5 МБ/с
Правило #1 | Порт 8443 | Динамика  | DL/UL 3 МБ/с, квота 100 МБ/10с, штраф 60с
Правило #2 | Порт 0    | Статика   | DL/UL 1 МБ/с  (fallback для всех остальных)
```

Порт `0` — wildcard, срабатывает для любого трафика, у которого нет точного совпадения. Точные порты всегда имеют приоритет.

## Белый список

IP из глобального белого списка (`/etc/shaffi/global-whitelist.txt`) полностью обходят шейпер. Синхронизируется с Fail2Ban, UFW и geo-block.

## Структура

```
modules/
  core/
    common.sh           — цвета, вывод, ввод, утилиты
    dependencies.sh     — авто-установка пакетов
  local/
    shaper.bpf.c        — eBPF программа (скомпилируется в .o)
    traffic_limiter.sh  — TUI меню шейпера
    shaffi_ctrl.py      — управление BPF-картами через bpftool
  security/
    whitelist_manager.sh — глобальный белый список
config/
  shaffi.conf           — переменные конфигурации
docs/
  Sheiper_trafik.md     — подробная документация
```

## Ручное управление

```bash
# Посмотреть активные правила
python3 modules/local/shaffi_ctrl.py --pin-dir /sys/fs/bpf/shaffi/maps rules

# Добавить статическое правило
python3 modules/local/shaffi_ctrl.py --pin-dir /sys/fs/bpf/shaffi/maps \
  set --rule-id 0 --mode 1 --ports 443 --down 5 --up 5

# Добавить динамическое правило
python3 modules/local/shaffi_ctrl.py --pin-dir /sys/fs/bpf/shaffi/maps \
  set --rule-id 1 --mode 2 --ports 8443 \
  --down 3 --up 3 --pen 0.5 --burst 100 --win 10 --pen-sec 60

# Статистика (топ IP)
python3 modules/local/shaffi_ctrl.py --pin-dir /sys/fs/bpf/shaffi/maps status

# Удалить правило
python3 modules/local/shaffi_ctrl.py --pin-dir /sys/fs/bpf/shaffi/maps \
  delete --rule-id 1
```
